What these flags represent

CCM lists CPU capability flags derived from CPUID. These are “the CPU can execute these opcodes”. Whether your software actually uses them depends on how it was compiled and whether it does runtime dispatch.
Flag Meaning (high-level)
SSE First generation Streaming SIMD Extensions (vector operations).
SSE2 Extends SSE with more integer and double-precision support; widely required by modern software.
SSE3 Incremental SIMD extensions (new horizontal ops, etc.).
SSSE3 Supplemental SSE3; adds common shuffle/byte operations.
SSE4.1 More integer/vector ops (blend, dot product, etc.).
SSE4.2 String/CRC-related ops and other additions.
AVX Advanced Vector Extensions (256-bit SIMD registers/instructions). Requires OS support to save/restore extended state.
AVX2 Extends AVX with 256-bit integer operations.
FMA Fused multiply-add (a*b+c in one instruction). Often paired with AVX.
BMI1 Bit Manipulation Instructions set 1 (bit scanning/extraction helpers).
BMI2 Bit Manipulation Instructions set 2 (e.g., PDEP/PEXT for bit deposit/extract).

Notes

  • These are capability flags (CPUID). Whether an app uses them depends on the compiler, runtime dispatch, and OS support.
  • For AVX/AVX2, the OS must enable XSAVE/XRESTORE support; this app checks OS support when detecting AVX.
  • ISA features matter beyond pure performance: wider SIMD state can increase context-switch overhead and some instruction mixes can influence turbo/frequency behavior.
A common pattern in performance libraries is: detect CPU features at startup (CPUID), then select the best implementation (scalar → SSE2 → AVX2, etc.). This is why you might see different performance on the same CPU depending on the exact build of an application.
